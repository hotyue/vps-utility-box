# 流量活跃模块 (Traffic Active)

本模块用于维持 VPS 的网络 IO 活跃度。脚本通过定期从官方镜像源下载随机大小的数据包，并在任务完成后立即清理，确保网络带宽有持续活动，防止因网络长期闲置被云服务商判定为资源浪费。

> **💡 部署提示**
> 杂货铺 v2.0+ 版本已进入全自动交互时代。请返回 [仓库根目录](../README.md) 使用 `deploy.sh` 进行全局一键部署，**无需手动单独下载运行本文件**。

## ⚙️ 运行逻辑与参数

在交互式部署时，您可以自由设定每次下载的数据量上下限和执行频率：

- **下载范围**: 推荐设定为每次任务随机下载 **20MB ~ 50MB** 的数据量。
- **自动清理**: 采用 `/tmp/vps_traffic_test.tmp` 作为临时缓存，下载完成后立即执行 `rm` 删除，实现 **0 磁盘残留**。
- **可靠流量源**: 默认采用 Debian 官方镜像站的大文件作为下载源，模拟合法的系统更新与维护流量，安全稳定。
- **日志记录**: 任务的启动时间、本次随机生成的目标下载量及完成状态均追加至统一日志 `/var/log/vps_maintenance.log`。

## 🔍 监控观察与流量预估

当本模块处于部署状态时，您可以通过以下方式监控网络活跃情况：

```bash
# 1. 实时查看网络活跃日志
tail -f /var/log/vps_maintenance.log | grep "[网络]"

# 2. 统计历史执行记录
grep "[网络] 任务完成" /var/log/vps_maintenance.log
```

## 流量消耗参考：

按每 10 分钟运行一次计算:

- 每日约产生 4.3GB - 7.2GB 下行流量。

- 月度累计流量约 130GB - 210GB。

请确保您的 VPS 流量配额充足。
