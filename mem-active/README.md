# 内存动态占用模块 (Memory Active)

本模块用于模拟 VPS 的内存使用活跃度。通过分配指定范围的随机内存并在规定时间内保持，防止 VPS 因内存长期处于极低占用状态而被云服务商回收。

> **💡 部署提示**
> 杂货铺 v2.0+ 版本已进入全自动交互时代。请返回 [仓库根目录](../README.md) 使用 `deploy.sh` 进行全局一键部署，**无需手动单独下载运行本文件**。

## ⚙️ 运行逻辑与参数

在交互式部署时，您可以自由设定以下参数。脚本每次被触发时，会在设定区间内随机申请内存资源：

- **依赖工具**: 底层调用 `stress-ng`（脚本执行时会自动检测并尝试静默安装）。
- **动态占用范围**: 推荐设定为 **100MB ~ 300MB** 的随机物理内存分配。
- **动态持续时长**: 推荐设定为每次任务持续 **100秒 ~ 150秒**。
- **静默挂起模式**: 核心逻辑引入了 `--vm-hang` 参数。分配完指定内存后，进程即进入休眠（Sleep）状态，**不产生高频内存读写，保证运行时 CPU 占用率趋近于 0**。
- **内存回收**: 任务倒计时结束后，进程会自动退出并瞬间释放所有占用的内存，不会引发系统 OOM (Out Of Memory)。

## 🔍 监控观察

当本模块处于部署状态时，您可以通过以下系统命令验证底层逻辑是否正常生效：

```bash
# 1. 实时查看物理内存占用变化 (每秒刷新一次)
watch -n 1 free -m

# 2. 专门过滤查看内存模块的运行日志
tail -f /var/log/vps_maintenance.log | grep "[内存]"
```
