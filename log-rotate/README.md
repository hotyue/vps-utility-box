# 日志管理模块 (Log Rotate)

本模块专为管理 `/var/log/vps_maintenance.log` 运行日志而设计。通过系统自带的 `logrotate` 工具，实现日志的每日自动轮转、压缩及过期清理，防止长时间运行维护脚本产生的日志占满 VPS 磁盘空间。

> **💡 部署提示**
> 杂货铺 v2.0+ 版本已进入全自动交互时代。请返回 [仓库根目录](../README.md) 使用 `deploy.sh` 进行全局一键部署，**无需手动下载与配置文件**。

## ⚙️ 配置说明

系统底层实际生效的配置文件 (`vps_maintenance`) 遵循以下核心规则：

| 配置项 | 功能描述 |
| :--- | :--- |
| **daily** | **每天**自动执行一次轮转检查。 |
| **rotate 7** | 仅保留最近 **7 份**历史日志，过期的日志会被自动物理删除。 |
| **copytruncate** | **先复制再清空**。在备份当前日志的同时清空原文件，确保正在写入日志的脚本不会因文件被移除而中断记录。 |
| **compress** | 对旧日志进行 **Gzip 压缩**，极大地节省存储空间。 |
| **missingok** | 如果日志文件不存在，则忽略错误，不产生系统报警。 |
| **notifempty** | 如果日志文件为空（字节为 0），则不执行轮转操作。 |

## 🔍 验证与维护

当您的 VPS 部署完毕后，可以通过以下命令手动触发轮转，以验证配置是否正确生效：

```bash
# 强制执行一次轮转测试
sudo logrotate -f /etc/logrotate.d/vps_maintenance

# 查看日志目录，确认是否生成了 .gz 压缩包
ls -lh /var/log/vps_maintenance*
```
